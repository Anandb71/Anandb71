## Founder @getarbor.dev

I am building **Arbor**, a graph-native code intelligence platform for AI coding agents and engineering teams. The bet: code assistants should reason over real program structure, not vibes wrapped in a vector search result.

My work sits at the intersection of deterministic code graphs, agent observability, local-first tooling, and polished product surfaces. I care about systems that can answer the important questions clearly: what changed, what depends on it, why did it happen, and what breaks next?

```text
deterministic core -> inspectable state -> fast feedback -> beautiful interface
```

## Working Style

- **Graphs first:** structure beats guesswork when agents need reliable context.
- **Local-first by default:** privacy, latency, and control are product features.
- **Observable agents:** every routing decision should leave useful evidence behind.
- **Interfaces with taste:** technical tools can be sharp, fast, and cinematic without becoming noisy.
